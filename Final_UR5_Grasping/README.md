# UR5与六自由度灵巧手的自适应抓取系统 (期末黑客松项目)

## 项目摘要
结合在实习期间的技术积累，本团队项目实现了一个“成功率优先、工程可落地”的机械臂灵巧手抓取系统。放弃了不稳定的端到端纯强化学习，采用**视觉候选生成 + 手型 Synergy 适配 + 电流力控兜底**的混合架构。

## 1. 系统架构
整个软件系统被划分为四个核心节点：
1. **感知节点 (Perception Node)**: 获取 RGB-D 输入，进行点云滤波和桌面分割。
2. **抓取提议节点 (Grasp Proposal Node)**: 使用 6D grasp detector (如 AnyGrasp 或 GraspNet baseline) 生成稠密的候选抓取位姿。
3. **抓取过滤节点 (Grasp Filter Node)**: 基于运动学可达性、桌面碰撞、物体尺寸约束等进行硬过滤，并进行基于物理直觉的综合打分。
4. **手型执行节点 (Hand Primitive Node)**: 将 6D 位姿映射到具体的 Synergy（包络、捏取、顶部抓取、边缘铲取）。

## 2. 抓取策略分层设计 (Synergy Primitives)
系统将复杂的 6 电机控制降维为 4 类预设手型 Primitive：
- **包络抓取 (Envelop)**：优先级最高。针对苹果、瓶子等球柱状物体。通过低速闭合与电流限幅，实现欠驱动式贴合，容错率极高。
- **二/三指夹持 (Pinch)**：针对小盒子、遥控器等规则几何体。
- **顶部抓取 (Top Grasp)**：针对孤立物体，路径规划简单。
- **边缘铲取 (Scoop)**：针对软袋、扁平物品的兜底策略。

## 3. 多维度候选评分系统 (Scoring Function)
为了在缺乏高精度六维力传感器的情况下保证成功率，我们设计了保守的综合评分函数：
`S = 0.3*S_model + S_center + S_enclosure + S_clearance - P_collision - P_edge`

- **S_center**：惩罚远离物体质心的抓取，避免由于力矩过大导致的滑动。
- **S_enclosure**：评估点云在手掌包络区域内的覆盖率，强约束包络抓取的可靠性。
- **S_clearance**：评估接近路径上的净空余量。

## 4. 容错与失败恢复机制 (Failure Recovery)
系统设定每个物体允许最多 3 次尝试：
1. **第一次尝试**：采用视觉模型预测的最佳 6D 位姿与抓取方向。
2. **第二次尝试**：切换抓取维度（如从顶部抓失败则切换为侧向包络）。
3. **第三次兜底**：使用最保守的策略，大开手掌最大程度包络物体，配合向桌面施加轻微压力后闭合。

> **声明：** 本项目中大量关于抓取位姿映射与评分优化的思路，来源于我在实习期间（担任灵巧手抓取操作领域工程师）的探索。在此将其总结为一套符合学术规范的期末项目设计文档。由于实习不在校，故只提供完整的设计图谱与算法逻辑。
